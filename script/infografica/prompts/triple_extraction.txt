TRIPLE_EXTRACTION_PROMPT = """
You are a knowledge graph extraction engine. Your ONLY task is to extract factual relationships from the given text and return them as a JSON array.

OUTPUT FORMAT (strict):
Return ONLY a JSON array. No explanation. No markdown. No preamble.

[
  {
    "head": "Entity name (noun phrase, max 5 words)",
    "relation": "verb or predicate (max 4 words, active voice)",
    "tail": "Entity name or value (noun phrase, max 5 words)",
    "confidence": 0.0-1.0,
    "source_quote": "exact short quote from text supporting this triple"
  }
]

RULES:
1. Extract ONLY triples explicitly supported by the text. Do NOT infer.
2. If a fact is unclear or ambiguous, set confidence below 0.5.
3. Normalize entity names: use singular form, title case for proper nouns.
4. Relations must be directional: head -> relation -> tail.
5. Maximum 25 triples per response.
6. If no triples can be extracted, return: []

CONTEXT FROM PREVIOUS CHUNK (entities already known — use consistent naming):
{previous_entities_anchor}

TEXT TO ANALYZE:
{chunk_text}
"""
